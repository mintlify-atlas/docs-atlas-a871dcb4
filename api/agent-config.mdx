Configuration constants that map agent names to their digest title patterns and target databases. Used by `check-upstream-status` and `write-agent-digest` workers.

## Constants

### VALID_AGENT_NAMES

Array of all recognized agent names in the system.

```typescript
export const VALID_AGENT_NAMES: string[];
```

**Current valid agents:**

```typescript
[
  "Inbox Manager",
  "Personal Ops Manager",
  "GitHub Insyncerator",
  "Client Repo Auditor",
  "Docs Librarian",
  "VEP Weekly Reporter",
  "Home & Life Watcher",
  "Template Freshness Watcher",
  "Time Log Auditor",
  "Client Health Scorecard",
  "Morning Briefing"
]
```

### AGENT_DIGEST_PATTERNS

Maps each agent name to an array of digest title patterns it can produce.

```typescript
export const AGENT_DIGEST_PATTERNS: Record<string, string[]>;
```

**Values:**

```typescript
{
  "Inbox Manager": ["Email Triage"],
  "Personal Ops Manager": ["Personal Triage"],
  "GitHub Insyncerator": ["GitHub Sync"],
  "Client Repo Auditor": ["Client Repo Audit"],
  "Docs Librarian": ["Docs Quick Scan", "Docs Cleanup Report"],
  "VEP Weekly Reporter": ["VEP Weekly Activity Report"],
```

<Note>
Agents with multiple patterns (like Docs Librarian) may produce different digest types depending on the operation mode.
</Note>

## Functions

### isValidAgentName

Checks if a given name is a recognized agent.

```typescript
function isValidAgentName(name: string): boolean
```

**Parameters:**
- `name` - Agent name to validate

**Returns:**
- `true` if the name exists in `VALID_AGENT_NAMES`
- `false` otherwise

**Example:**

```typescript
import { isValidAgentName } from "./agent-config.js";

isValidAgentName("Inbox Manager"); // true
isValidAgentName("Unknown Agent"); // false
```

### getDefaultDigestType

Returns the default (first) digest pattern for an agent.

```typescript
function getDefaultDigestType(agentName: string): string
```

**Parameters:**
- `agentName` - Agent name (should be valid)

**Returns:**
- First digest pattern from `AGENT_DIGEST_PATTERNS`
- Returns the agent name itself if no patterns are configured

**Example:**

```typescript
import { getDefaultDigestType } from "./agent-config.js";

getDefaultDigestType("Inbox Manager"); // "Email Triage"
getDefaultDigestType("Docs Librarian"); // "Docs Quick Scan"
getDefaultDigestType("GitHub Insyncerator"); // "GitHub Sync"
```

<Note>
This function is used by `write-agent-digest` to generate page titles. The first pattern in the array is considered the primary/default digest type for each agent.
</Note>

## Usage Examples

### Validating Agent Names

```typescript
import { isValidAgentName, VALID_AGENT_NAMES } from "./agent-config.js";

function validateInput(agentName: string): void {
  if (!isValidAgentName(agentName)) {
    throw new Error(
      `Invalid agent_name: "${agentName}". Must be one of: ${VALID_AGENT_NAMES.join(", ")}`
    );
  }
}
```

### Getting Target Database

```typescript
import { AGENT_TARGET_DB } from "./agent-config.js";
import type { TargetDatabase } from "./types.js";

function getTargetDatabase(agentName: string): TargetDatabase {
  const db = AGENT_TARGET_DB[agentName];
  if (!db) {
    throw new Error(`No target database configured for ${agentName}`);
  }
  return db;
}
```

### Finding Digest Pages

```typescript
import { AGENT_DIGEST_PATTERNS } from "./agent-config.js";

function buildTitleFilters(agentName: string) {
  const patterns = AGENT_DIGEST_PATTERNS[agentName] ?? [];
  
  // Create Notion filter for any matching digest pattern
  return {
    or: patterns.map(pattern => ({
      property: "Name",
      title: { contains: pattern }
    }))
  };
}
```

### Building Page Titles

```typescript
import { getDefaultDigestType } from "./agent-config.js";

function buildDigestTitle(agentName: string, emoji: string, timestamp: string): string {
  const digestType = getDefaultDigestType(agentName);
  return `${emoji} ${digestType} — ${timestamp}`;
}

// Example:
buildDigestTitle("Inbox Manager", "📧", "2026-03-04");
// Returns: "📧 Email Triage — 2026-03-04"
```

## Adding New Agents

To add a new agent to the system:

1. Add the agent name and digest pattern(s) to `AGENT_DIGEST_PATTERNS`
2. Add the target database to `AGENT_TARGET_DB`
3. `VALID_AGENT_NAMES` is automatically derived from the patterns object

```typescript
export const AGENT_DIGEST_PATTERNS: Record<string, string[]> = {
  // ... existing agents ...
  "New Agent Name": ["New Digest Pattern"],
};

export const AGENT_TARGET_DB: Record<string, TargetDatabase> = {
  // ... existing agents ...
  "New Agent Name": "docs", // or "home_docs"
};
```
