Core type definitions used across all workers for status management, agent digests, and handoff markers.

## Status Types

### StatusType

Defines the type of status being reported.

```typescript
export type StatusType = "sync" | "snapshot" | "report" | "heartbeat";
```

- `sync` - Regular synchronization operations
- `snapshot` - Point-in-time snapshot operations
- `report` - Report generation
- `heartbeat` - Keep-alive signal with no actionable items

### StatusValue

Indicates the outcome of an operation.

```typescript
export type StatusValue = "complete" | "partial" | "failed" | "full_report" | "stub";
```

- `complete` - Operation finished successfully
- `partial` - Operation completed with some issues
- `failed` - Operation failed
- `full_report` - Full report generated (report type only)
- `stub` - Placeholder/stub generated (report type only)

### TargetDatabase

Specifies which Notion database to write to.

```typescript
export type TargetDatabase = "docs" | "home_docs";
```

- `docs` - Professional documentation database
- `home_docs` - Personal home & life database

### TaskPriority

Task priority levels with emoji indicators.

```typescript
export type TaskPriority = "🔴 High" | "🟡 Medium" | "🟢 Low";
```

## Core Interfaces

### FlaggedItem

Items requiring attention or follow-up.

```typescript
export interface FlaggedItem {
  description: string;
  task_link?: string;
  no_task_reason?: string;
}
```

<Expandable title="Field Details">
- `description` - What needs attention
- `task_link` - Optional Notion task URL if created
- `no_task_reason` - Why no task was created (if applicable)
</Expandable>

### TaskRef

Reference to a Notion task.

```typescript
export interface TaskRef {
  name: string;
  notion_url: string;
}
```

### ActionsTaken

Summary of tasks created or updated during a run.

```typescript
export interface ActionsTaken {
  created_tasks: TaskRef[];
  updated_tasks: TaskRef[];
}
```

### NeedsReview

Items flagged for manual review.

```typescript
export interface NeedsReview {
  description: string;
}
```

### Escalation

Handoff information when escalating to another agent.

```typescript
export interface Escalation {
  escalated_to: string;
  escalation_reason: string;
  escalation_owner: string;
  handoff_complete: boolean;
}
```

<Expandable title="Field Details">
- `escalated_to` - Target agent name
- `escalation_reason` - Why this is being escalated
- `escalation_owner` - Who owns the escalation
- `handoff_complete` - Whether handoff marker was created
</Expandable>

## Worker Input/Output Types

### WriteAgentDigestInput

Input for creating agent digest pages.

```typescript
export interface WriteAgentDigestInput {
  agent_name: string;
  agent_emoji: string;
  status_type: StatusType;
  status_value: StatusValue;
  run_time_chicago: string;
  scope: string;
  input_versions: string;
  flagged_items: FlaggedItem[];
  actions_taken: ActionsTaken;
  summary: string;
  needs_review: NeedsReview[];
  escalations: Escalation[];
  target_database: TargetDatabase;
  doc_type: string;
  client_relation_ids?: string[];
  project_relation_ids?: string[];
}
```

<Expandable title="Field Details">
- `agent_name` - Must match VALID_AGENT_NAMES
- `agent_emoji` - Emoji for page title (e.g., "📧")
- `status_type` - Type of operation
- `status_value` - Operation outcome
- `run_time_chicago` - ISO timestamp for Chicago timezone formatting
- `scope` - What was processed (e.g., "All inbox items")
- `input_versions` - Version info for inputs
- `flagged_items` - Items needing attention
- `actions_taken` - Tasks created/updated
- `summary` - Brief summary of the run
- `needs_review` - Items for manual review
- `escalations` - Handoffs to other agents
- `target_database` - Where to write the digest
- `doc_type` - Document type for the page
- `client_relation_ids` - Optional client relations (32-char hex)
- `project_relation_ids` - Optional project relations (32-char hex)
</Expandable>

### WriteAgentDigestOutput

Result of digest creation.

```typescript
export type WriteAgentDigestOutput =
  | {
      success: true;
      page_url: string;
      page_id: string;
      title: string;
      is_error_titled: boolean;
      is_heartbeat: boolean;
    }
  | { success: false; error: string };
```

<Expandable title="Success Response">
- `success: true` - Operation succeeded
- `page_url` - Full Notion URL to created page
- `page_id` - 32-character Notion page ID
- `title` - Generated page title
- `is_error_titled` - Whether title indicates an error state
- `is_heartbeat` - Whether this is a heartbeat digest
</Expandable>

### CheckUpstreamStatusInput

Input for checking upstream agent status.

```typescript
export interface CheckUpstreamStatusInput {
  agent_name: string;
  max_age_hours?: number;
  require_current_cycle?: boolean;
}
```

<Expandable title="Field Details">
- `agent_name` - Agent to check (must match VALID_AGENT_NAMES)
- `max_age_hours` - Optional maximum age threshold (default: 24)
- `require_current_cycle` - Whether to require current cycle digests
</Expandable>

### UpstreamStatus

Possible upstream agent status values.

```typescript
export type UpstreamStatus =
  | "complete"
  | "partial"
  | "failed"
  | "full_report"
  | "stub"
  | "not_found"
  | "stale"
  | "unknown";
```

### CheckUpstreamStatusOutput

Detailed upstream status information.

```typescript
export interface CheckUpstreamStatusOutput {
  found: boolean;
  agent_name: string;
  status: UpstreamStatus;
  status_type: StatusType | null;
  run_time: string | null;
  run_time_age_hours: number | null;
  is_stale: boolean;
  is_heartbeat: boolean;
  is_error_titled: boolean;
  page_url: string | null;
  page_id: string | null;
  degraded: boolean;
  data_completeness_notice: string;
}
```

<Expandable title="Field Details">
- `found` - Whether any digest was found
- `agent_name` - Agent that was checked
- `status` - Parsed status from digest
- `status_type` - Type of status (sync/snapshot/report)
- `run_time` - Formatted run time string
- `run_time_age_hours` - Hours since the digest was created
- `is_stale` - Whether digest exceeds max_age_hours
- `is_heartbeat` - Whether this is a heartbeat
- `is_error_titled` - Whether title indicates error
- `page_url` - Notion URL if found
- `page_id` - Page ID if found
- `degraded` - Whether service is degraded (partial/failed/stale)
- `data_completeness_notice` - Human-readable status message
</Expandable>

### CreateHandoffMarkerInput

Input for creating handoff markers.

```typescript
export interface CreateHandoffMarkerInput {
  source_agent: string;
  target_agent: string;
  escalation_reason: string;
  source_digest_url: string;
  create_task: boolean;
  task_priority?: TaskPriority;
  client_relation_ids?: string[];
  project_relation_ids?: string[];
}
```

<Expandable title="Field Details">
- `source_agent` - Agent creating the handoff
- `target_agent` - Agent receiving the handoff
- `escalation_reason` - Why this is being escalated
- `source_digest_url` - URL to source digest page
- `create_task` - Whether to create a task
- `task_priority` - Priority if creating task
- `client_relation_ids` - Optional client relations
- `project_relation_ids` - Optional project relations
</Expandable>

### CreateHandoffMarkerOutput

Result of handoff marker creation.

```typescript
export type CreateHandoffMarkerOutput =
  | {
      success: true;
      handoff_block: string;
      task_created: boolean;
      task_url: string | null;
      task_id: string | null;
      duplicate_prevented: boolean;
      existing_task_url: string | null;
      escalation_capped: boolean;
      needs_manual_review: boolean;
    }
  | { success: false; error: string };
```

<Expandable title="Success Response">
- `success: true` - Operation succeeded
- `handoff_block` - Formatted handoff marker text
- `task_created` - Whether a new task was created
- `task_url` - Task URL if created
- `task_id` - Task ID if created
- `duplicate_prevented` - Whether duplicate was detected
- `existing_task_url` - URL to existing task if duplicate
- `escalation_capped` - Whether escalation limit was reached
- `needs_manual_review` - Whether manual review is required
</Expandable>

## Usage Examples

### Creating a Digest

```typescript
import type { WriteAgentDigestInput } from "./types.js";

const input: WriteAgentDigestInput = {
  agent_name: "Inbox Manager",
  agent_emoji: "📧",
  status_type: "sync",
  status_value: "complete",
  run_time_chicago: new Date().toISOString(),
  scope: "All inbox items",
  input_versions: "v1.2.3",
  flagged_items: [],
  actions_taken: { created_tasks: [], updated_tasks: [] },
  summary: "Processed 42 emails",
  needs_review: [],
  escalations: [],
  target_database: "docs",
  doc_type: "Agent Digest"
};
```

### Checking Upstream Status

```typescript
import type { CheckUpstreamStatusInput } from "./types.js";

const input: CheckUpstreamStatusInput = {
  agent_name: "GitHub Insyncerator",
  max_age_hours: 24,
  require_current_cycle: true
};
```
